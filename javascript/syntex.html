    7. JavaScript Identifiers / Names

      1. Identifiers are JavaScript names. Identifiers are used to name variables and keywords, and functions.
      2. The rules for legal names are the same in most programming languages.
      3. A JavaScript name must begin with: A letter (A-Z or a-z) A dollar sign ($) Or an underscore (_) Subsequent characters may be letters, digits, underscores, or dollar signs.
      4. Note Numbers are not allowed as the first character in names. This way JavaScript can easily distinguish identifiers from numbers.

    8. JavaScript is Case Sensitive

      1. All JavaScript identifiers are case sensitive.
      2. The variables lastName and lastname, are two different variables: JavaScript does not interpret LET or Let as the keyword let.

    9. JavaScript and Camel Case

      1. Historically, programmers have used different ways of joining multiple words into one variable name

      2. Hyphens:
        first-name, last-name, master-card, inter-city. Hyphens are not allowed in JavaScript. They are reserved for subtractions.
      3. Underscore:
        first_name, last_name, master_card, inter_city.
      4. Upper Camel Case (Pascal Case):
        FirstName, LastName, MasterCard, InterCity.
      5. Lower Camel Case:
        JavaScript programmers tend to use camel case that starts with a
      6. lowercase letter: firstName, lastName, masterCard, interCity.
